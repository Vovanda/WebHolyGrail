# Content framing: «лоза — только без своих линий»

> Источник правды по API — JSDoc в `sites/<site>/src/client/src/blocks/decor/ContentFrame.tsx`. Эта страница — про **принцип** выбора декора, с примерами и причинами.

## TL;DR

`ContentFrame decor="vines"` ставим **только когда у внутреннего контента нет собственных боковых линий**. Иначе двойная подсветка = визуальный шум, бренд-впечатление портится.

```tsx
// ✅ OK — голый текст, лоза уместна
<ContentFrame side="both" decor="vines">
  <Prose ... />
</ContentFrame>

// ✅ OK — цитата со своим border-l, лозу выключили
<ContentFrame side="none">
  <Quote ... />
</ContentFrame>
```

## Что такое `ContentFrame`

Универсальный wrapper-контейнер контентной секции:

1. **Единая ширина** — `max-w-content` (одна полоса для всех секций сайта).
2. **Опциональные декор-границы** по сторонам — `decor × side`:
   - `decor`: `lines` (generic) или `vines` (niche-виньетка — декоративные лозы/орнамент из изображений)
   - `side`: `left | right | both | none`

См. JSDoc в самом компоненте — там полная таблица вариантов и примеры.

## Принцип «лоза — только без своих линий»

### Правило

Если у внутреннего блока **уже есть собственный боковой акцент** — `decor` ставим `'none'`. Лозы / линии только когда контент «голый» по бокам.

### Чек-лист перед `decor="vines"`

- [ ] У блока внутри **нет** `border-l-*` / `border-r-*`?
- [ ] У блока внутри **нет** края слайдера / галереи на 100% ширины?
- [ ] У блока внутри **нет** линии цитаты / accent-bar / divider'а по вертикали?

Все три «нет» → `vines` уместны. Любое «да» → `decor='none'` (или вообще без `ContentFrame`).

### Почему — детально

#### 1. Двойная подсветка = визуальный шум

Лоза + border-l цитаты создают «двойную рамку» — взгляд цепляется, читать сложнее, бренд-впечатление портится. Один акцент на одной вертикали = единый ритм.

```
До:                                  После:
║
║  ▸ цитата с border-l               │ ▸ цитата с border-l
║  ▸ цитата с border-l               │ ▸ цитата с border-l
║
↑ лоза                               ↑ только border цитаты
↑ + border цитаты = две линии        = один акцент, чисто
```

#### 2. Лоза — декоративная виньетка, не контейнер

Лоза оформляет «пустые» края секции (когда контент маленький / центрирован). Когда у блока есть свой край (галерея занимает всю ширину, цитата с border-l) — лоза становится **фоном**, не виньеткой, выглядит лишней.

#### 3. Mobile-проблема

На узких экранах лоза часто скрывается / уезжает за viewport. Если у внутреннего блока есть свой боковой акцент — лучше его показать, чем сжатую лозу.

## Когда применять (примеры)

### ✅ `decor="vines"` — голый контент

- **Prose** (только текст, без своих рамок)
- **Timeline** (содержимое центрировано, без боковин)
- **Hero**-плашки без `border-l`
- **AchievementBanner** в нейтральном `accent='neutral'` (без цветной рамки)

### ❌ `decor="none"` — есть свой акцент

- **Quote** с `border-l-accent`
- **Carousel / BannerSlider** — занимают всю ширину
- **AchievementBanner** с цветным `accent='success'/'info'/...` (своя рамка)
- **CertifiedNotice** с боковой подсветкой
- **PairCardGallery** — фото на всю ширину контента

## Когда `lines` вместо `vines`

`lines` — тонкие полупрозрачные вертикальные акценты, без иллюстраций. Generic для любого сайта.

Использовать когда:

- бренд-система сайта **не предусматривает** виньетки (минимализм, SaaS, корпоратив)
- секция должна быть **более сдержанной** (официальные документы, форма)

```tsx
<ContentFrame side="both" decor="lines">
  <FormBlock ... />
</ContentFrame>
```

## Связь с правилами фреймворка

- **R2 (токены).** Цвет и непрозрачность лоз — через `--decor-vine-*` переменные в `tokens.css`, не hex.
- **R5++ (функциональное имя).** `ContentFrame` — функциональный, не доменный («не VeoVineFrame»). На любом сайте.
- **R6 (без догм).** Лозу можно вытащить из `globals.css` и заменить (на другом сайте они не используются).
